Gridded Binary (GRIB) Data

Gridded Binary (GRIB) is a common data format used in meteorology to store and disseminate both, historical and forecasted, weather information. In fact it is the format recommended by the World Meteorological Organization (WMO). One example of GRIB data that immediately comes to mind is the NOAA Global Forecast System (GFS) data generated using the weather forecast model by the National Centers for Environmental Prediction (NCEP).

How to download and read GRIB data?

There are several sources and ways to download and read the GRIB data. One of the common source for the historical or near real time data from GFS is the S3 bucket on Amazon AWS, one of NOAA’s Big Data partner. Couple of of the common Python packages to download and work with this data, as far as I know, are pygrib and Herbie. In this post, I explore the functionality of Herbie to work with the GFS data.

Installing Herbie

According to the package documentation, Herbie requires curl and Python version 3.8+ and the following packages: requests, pandas, xarray, and cfgrib. Other optional packages include matplotlib and cartopy. I already have a Python 3.9.7 installed on my system. Let’s install the package into a dedicated conda environment using [this YAML file] provided on the package GitHub page. To do this we run the following commands in the conda terminal:

# download YAML file
curl -O https://github.com/blaylockbk/Herbie/raw/main/environment.yml

# Create new environment using the specifications provided in the YAML file.
conda env create -f environment.yml
